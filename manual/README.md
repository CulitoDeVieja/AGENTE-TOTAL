# Manual de recetas

El Manual es donde los agentes guardan **cómo hacer bien las cosas**. Está dividido en dos:

- `general/` → recetas que usan todos los capataces (login, tests, deploy, etc.)
- `capataces/` → recetas específicas por especialidad (frontend, backend, móvil, datos, etc.)

Cada receta es un archivo Markdown corto con:

1. **Qué problema resuelve** (1-2 líneas)
2. **Cuándo usarla** (en qué tipo de tarea aplica)
3. **Pasos** (breves y concretos)
4. **Ejemplo** (opcional)
5. **Notas** (advertencias, errores comunes)

En la Fase 1 (auto-construcción) el sistema va a cargar 20-30 recetas base acá adentro. Por ahora las carpetas están vacías a propósito.

## Quién edita este manual

- Los **obreros** pueden proponer nuevas recetas cuando aprenden algo útil.
- El **bibliotecario** revisa propuestas diariamente y te manda un resumen.
- **Vos aprobás** los cambios grandes.
